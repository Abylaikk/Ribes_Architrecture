Ribes — Marketplace & Runtime for Local AI

A distribution platform and execution runtime for AI agents. Developers publish agents to a cloud catalog; users install them in one click and run them on their own hardware — no cloud inference cost, no data leaving the machine. This document maps every layer, language, framework and data flow of the product.

03Frontend

React + TypeScript, matte industrial theme
React 18 + Vite

Component SPA built with Vite. Production bundle is embedded into the Go binary via go:embed, so the whole platform ships as one file.

TypeScript

Strict types mirror the backend models one-to-one (lib/types.ts), keeping API contracts honest end to end.

Zustand stores

session, cart, downloads, social — reactive state without boilerplate.

Tailwind CSS

Matte dark design tokens: graphite surfaces, hairline borders, one muted purple accent. Flat fills, no gradients, no emoji icons.

Bridge layer

lib/bridge.ts abstracts the runtime: in the browser it calls the platform API; in the native window it drives the local execution engine. Same UI, both modes.

Realtime + WebCrypto

Auto-reconnecting WebSocket for presence/chat/events; WebCrypto verifies every downloaded chunk's SHA-256 in-browser.

04Backend

One static Go binary, layered by package
internal/api

HTTP surface: net/http.ServeMux (Go 1.22 method+path patterns), JWT auth middleware, CORS, WS upgrade. REST for catalog, identity, licensing, payments, social, dev center.

internal/store

The database layer. SQL written once in portable form; a rebind step turns ? into $N for PostgreSQL. Same queries drive both profiles. Additive idempotent migrations.

internal/ws

Realtime hub over gorilla/websocket: one socket per session, presence, direct messages, notifications. Event bus is Redis Pub/Sub in production, in-process for a single node.

internal/artifacts

Splits agent packages into 4 MiB content-addressed chunks, SHA-256 per chunk, dedup across versions. Serves them with ranged, resumable HTTP.

internal/security

Upload gate: product contract (manifest + runnable entrypoint present) and a heuristic malware scan of package sources. Rejects with explicit findings.

internal/local

Embedded execution engine (desktop profile): install → launch → stop (kill-tree) → uninstall of agent processes, plus disk-usage stats.

05Data layer

Two profiles, one SQL codebase
PostgreSQL

Production source of truth: catalog, identities, licenses, the payments ledger, reviews, social graph, builds.

SQLite (dev)

Zero-config embedded DB for single-node/MVP. WAL mode. Same SQL as PostgreSQL.

Redis

Pub/Sub bus so N horizontally-scaled Go instances fan out chat/presence/catalog events consistently.

S3 / Spaces + CDN

Artifact origin for multi-GB packages. Download plans point to presigned CDN URLs; SHA-256 stays the integrity anchor.

A separate SQLite database on the client (%LOCALAPPDATA%\Ribes\library.db) is the offline source of truth for what is installed on that machine — no provisioning required.

06Desktop shell, installer & SDK

Native runtime and the developer contract
Tauri 2 core (Rust)

Async launcher daemon on Tokio: hardware probe (sysinfo, nvml), resumable chunked downloads (reqwest, sha2), SQLite library state (rusqlite), Job-Object kill-tree.

Native window (Go)

The Go binary can also host its own WebView2 window (go-webview2) — one process, own taskbar icon and title, single-instance.

Ribes SDK (Python)

Thin bridge client speaking JSON-RPC 2.0 over stdio / named pipe / UDS. Agents declare a ribes.manifest.json (JSON Schema) and raise the bridge from an entrypoint.

SetupRibes.exe (installer)

Graphical WebView2 wizard in Go. The platform binary travels gzip-compressed inside it; installs per-user (no admin), verifies SHA-256, creates shortcuts. Also a --silent unattended mode.

Bridge protocol

Agent → core events: ready, status, progress, metric, log, output. Core → agent: shutdown, cancel, request_input.

07Key data flows

How the product actually moves
Install & launch pipeline
1
License gate

Backend verifies ownership, returns a signed download-plan: chunk list, SHA-256, CDN URLs.

2
Verified download

Each chunk fetched via ranged HTTP; SHA-256 checked before acceptance. Live progress over WebSocket.

3
Install

Package unpacked into the managed library; real size recorded; Desktop shortcut created targeting Ribes --run-agent <id>.

4
Launch / stop / uninstall

Platform spawns and supervises the process; graceful shutdown then kill-tree; uninstall removes files, shortcut and process.

Publish, pay & realtime
A
Publish

Dev uploads a package → security scan → moderation → catalog_changed fanned out to all clients.

B
Payment

Wallet-first checkout; on a paid sale the developer is credited 70% instantly into the transaction ledger, 30% platform commission.

C
Realtime

Chat / presence / notifications published to Redis → fanned out over WebSocket to every connected client.

D
Recommendations

Scored from library affinity + friends' libraries + rating + the local hardware gate.

09Security model

Trust boundaries & integrity
Identity
  • bcrypt password hashing
  • JWT HS256 sessions (30 days)
  • OAuth (Google / GitHub) with persistent accounts
Artifact integrity
  • Content-addressed 4 MiB chunks
  • SHA-256 verified before accept
  • zip-slip guards on unpack
Upload gate
  • Product contract: runnable entrypoint required
  • Heuristic malware scan of sources
  • Explicit rejection findings
Execution isolation
  • Process-level sandbox + scoped env
  • Kill-tree guarantees VRAM release
  • Egress denied by default (manifest permissions)
Payments
  • Only card last-4 stored; PAN to provider
  • Every movement recorded in the ledger
  • Separate developer payout card
Privacy by design
  • Inference runs locally — prompts never leave the device
  • CSP on the agent webview
